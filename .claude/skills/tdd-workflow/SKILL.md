# TDD Workflow

テスト駆動開発（TDD）のRed-Green-Refactoringサイクルに従って実装を進める。

## 対象

$ARGUMENTS

## 制約

- **テスト実行は必ず `api` コンテナ内で行う**
- 実装コードを書く前に、必ずテストコードを先に作成する
- 各フェーズの完了条件を満たしてから次のフェーズに進む

## テスト実行

テストの実行には `test-runner` サブエージェントを使用する。各フェーズでテスト実行が必要な場合、`test-runner` に対象パッケージとテスト関数名を渡して実行を委譲すること。

## フェーズ

### 1. Red（テスト作成・失敗確認）

1. 実装したい振る舞いを明確にする
2. テストコードを作成する（`_test.go` ファイル）
3. `test-runner` サブエージェントでテストを実行し、**失敗（FAIL）すること**を確認する

**完了条件**: `test-runner` から FAIL の報告を受けること。テストがpassしてしまった場合は、テストの内容を見直す。

### 2. Green（最小限の実装）

1. テストをpassさせる**最小限**の実装コードを書く
2. `test-runner` サブエージェントでテストを実行し、**成功（PASS）すること**を確認する

**完了条件**: `test-runner` から PASS の報告を受けること。この段階ではコードの美しさや効率は考慮しない。

### 3. Refactoring（リファクタリング）

1. コードの重複排除、命名改善、構造の整理を行う
2. `test-runner` サブエージェントでテストを実行し、**引き続き成功（PASS）すること**を確認する

**完了条件**: リファクタリング後も `test-runner` から PASS の報告を受けること。

## サイクルの繰り返し

上記3フェーズを1サイクルとして、必要な振る舞いが全て実装されるまで繰り返す。
次のサイクルに進む前に、現在のサイクルが完了していることを確認する。
